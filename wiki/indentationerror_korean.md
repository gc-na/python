# 파이썬 IndentationError: 들여쓰기 오류에 대한 완벽 가이드

## 개요
`IndentationError`는 파이썬에서 코드 블록의 들여쓰기가 잘못 되었을 때 발생하는 오류입니다. 이 오류는 파이썬의 문법 규칙에 따라 코드의 구조를 명확히 하기 위해 필요합니다.

## 문서화
`IndentationError`는 파이썬 프로그램에서 코드의 들여쓰기가 일관되지 않거나, 잘못된 위치에 있을 때 발생합니다. 파이썬은 코드 블록을 정의하기 위해 중괄호 대신 들여쓰기를 사용하기 때문에, 들여쓰기가 잘못되면 코드의 흐름을 이해할 수 없게 됩니다.

### 목적
이 오류는 코드의 가독성을 높이고, 프로그램의 논리적 구조를 명확하게 유지하기 위해 필수적입니다. 

### 사용법
코드 블록(예: 함수, 루프, 조건문 등)은 반드시 동일한 수준으로 들여쓰기를 해야 합니다. 파이썬에서는 일반적으로 4개의 스페이스(또는 1개의 탭)를 들여쓰기 단위로 사용합니다.

## 예제
### 올바른 예제
```python
def greet(name):
    print("안녕하세요, " + name + "님!")

greet("홍길동")
```

### 잘못된 예제
```python
def greet(name):
 print("안녕하세요, " + name + "님!")  # 잘못된 들여쓰기
    print("반갑습니다!")  # 불일치하는 들여쓰기
```
위의 코드에서 두 번째 `print` 문은 첫 번째 `print` 문과 들여쓰기가 다릅니다. 이로 인해 `IndentationError`가 발생합니다.

## 설명
`IndentationError`는 다음과 같은 경우에 발생할 수 있습니다:

1. **불일치하는 들여쓰기**: 코드 블록 내에서 들여쓰기가 일관되지 않은 경우.
2. **들여쓰기 누락**: 블록을 시작했지만 그에 대한 들여쓰기가 없는 경우.
3. **혼합된 들여쓰기**: 스페이스와 탭을 혼합하여 사용하는 경우.

이러한 오류를 피하기 위해서는 다음과 같은 팁을 따르는 것이 좋습니다:

- 코드 에디터에서 스페이스 또는 탭만 사용하도록 설정합니다.
- 함수, 조건문, 루프 등의 시작 부분에 대해 일관된 들여쓰기를 유지합니다.
- 코드 작성 후 항상 오류 메시지를 확인하고, 해당 줄에서 들여쓰기를 점검합니다.
