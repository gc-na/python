# الاستثناءات في بايثون: فهم وإدارة الأخطاء

## ملخص
في بايثون، تُعتبر الاستثناءات آلية قوية للتعامل مع الأخطاء. توفر هذه الميزة وسيلة لتنظيم الكود والتعامل مع الحالات غير المتوقعة بطريقة آمنة وفعالة.

## الوثائق
### الغرض
الاستثناءات هي أحداث غير عادية تحدث أثناء تنفيذ البرنامج. بدلاً من توقف البرنامج عند حدوث خطأ، يمكن استخدام الاستثناءات لإدارة الأخطاء بطريقة محسوبة.

### الاستخدام
تستخدم الاستثناءات في بايثون عبر الكتل `try` و`except`. تتيح لك هذه البنية محاولة تنفيذ كود معين والتقاط أي استثناء يحدث، مما يسمح لك بمعالجة الأخطاء دون إنهاء البرنامج.

### التفاصيل
- **try**: تُستخدم لتحديد الكود الذي قد يُسبب حدوث استثناء.
- **except**: تُستخدم لالتقاط ومعالجة الاستثناءات.
- **finally**: كتلة اختيارية تُنفذ دائمًا، سواء حدث استثناء أم لا.
- **raise**: تُستخدم لإثارة استثناء جديد.

## أمثلة
### مثال 1: استخدام `try` و`except`
```python
try:
    result = 10 / 0
except ZeroDivisionError:
    print("لا يمكن القسمة على صفر!")
```

### مثال 2: استخدام `finally`
```python
try:
    file = open("example.txt", "r")
    content = file.read()
finally:
    file.close()
```

### مثال 3: استخدام `raise`
```python
def check_age(age):
    if age < 0:
        raise ValueError("العمر لا يمكن أن يكون سالبًا")
        
try:
    check_age(-1)
except ValueError as e:
    print(e)
```

## الشرح
تعتبر الاستثناءات من الأدوات الأساسية في برمجة بايثون، لكن هناك بعض النقاط التي يجب الانتباه إليها:
- **عدم التقاط جميع الاستثناءات**: يُفضل التقاط استثناءات محددة بدلاً من التقاط جميعها باستخدام `except Exception:`.
- **الإفراط في استخدام الاستثناءات**: يمكن أن يؤدي استخدام الاستثناءات بشكل مفرط إلى جعل الكود أقل وضوحًا وأبطأ.
- **الاستثناءات المخصصة**: يمكن إنشاء استثناءات خاصة بك عن طريق وراثة `Exception`، مما يوفر طريقة لتنظيم الأخطاء بشكل أفضل.

## ملخص جملة واحدة
الاستثناءات في بايثون هي آلية قوية للتعامل مع الأخطاء، مما يسمح بكتابة كود أكثر مرونة وقابلية للصيانة.
