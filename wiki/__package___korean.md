# Python의 __package__ 속성: 패키지 관리의 기초

## 개요
`__package__`는 Python 모듈이 속한 패키지의 이름을 나타내는 속성입니다. 이 속성은 모듈이 어떤 패키지에 포함되어 있는지를 확인하고, 상대 경로 기반의 모듈 임포트를 가능하게 합니다.

## 문서화
### 목적
`__package__` 속성은 모듈이 속한 패키지를 명시하여, 그 패키지의 구조를 이해하고 상대 경로로 다른 모듈을 임포트하는 데 도움을 줍니다.

### 사용법
- `__package__`는 모듈이 실행될 때 자동으로 설정됩니다.
- 패키지 내에서 다른 모듈을 임포트할 때, 이 속성을 사용하여 모듈의 경로를 설정할 수 있습니다.

### 세부 사항
- `__package__`는 문자열로, 모듈이 속한 패키지의 이름을 담고 있습니다.
- 모듈이 최상위(즉, 패키지에 속하지 않음)일 경우, `__package__`는 빈 문자열이 됩니다.
- 상대 임포트를 사용하려면 `__package__`가 올바르게 설정되어 있어야 합니다.

## 예제
```python
# mypackage/__init__.py
__package__ = "mypackage"

# mypackage/mymodule.py
print(__package__)  # 출력: mypackage

# mypackage/submodule.py
from . import mymodule  # 상대 임포트
```

## 설명
- `__package__` 속성을 직접 수정하는 것은 권장되지 않습니다. 이 속성은 Python이 모듈을 로드할 때 자동으로 설정됩니다.
- 패키지 구조를 이해하지 못하면 상대 임포트가 실패할 수 있습니다. 따라서 패키지의 계층 구조를 잘 이해하고 있어야 합니다.
- `__package__`가 빈 문자열인 경우, 상대 경로 임포트는 동작하지 않으므로 주의해야 합니다.

## 한 줄 요약
`__package__`는 Python 모듈이 속한 패키지의 이름을 나타내며, 상대 경로 임포트를 가능하게 합니다.
