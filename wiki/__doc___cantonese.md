# Python 中的 __doc__ 屬性：完整指南

## 概述
在 Python 中，`__doc__` 是一個特殊的屬性，用於存儲對象的文檔字符串（docstring），提供關於對象的描述和用途的資訊。

## 文檔
`__doc__` 屬性可以被用來獲取任何 Python 對象（如函數、類或模組）的文檔字符串。文檔字符串通常是用三個引號包圍的字符串，位於對象定義的第一行。這樣的設計使得開發者能夠輕鬆地為其代碼添加說明，並且能夠在需要時快速檢索這些信息。

### 用法
使用 `__doc__` 屬性非常簡單。只需在對象後加上 `.__doc__` 即可獲取該對象的文檔字符串。例如：

```python
def my_function():
    """這是一個示範函數。"""
    pass

print(my_function.__doc__)
```

這段代碼將輸出函數 `my_function` 的文檔字符串。

### 詳細資訊
- `__doc__` 的返回值是對象的文檔字符串，若該對象沒有文檔字符串，則返回 `None`。
- `__doc__` 可以用於類、函數、模組和方法等多種對象。
- 使用文檔字符串的好處在於，可以通過內建的 `help()` 函數或其他文檔生成工具，輕鬆獲取和顯示對象的描述。

## 示例
以下是一些使用 `__doc__` 的基本例子：

### 1. 函數的文檔字符串
```python
def add(a, b):
    """返回兩數的和。"""
    return a + b

print(add.__doc__)  # 輸出：返回兩數的和。
```

### 2. 類的文檔字符串
```python
class Dog:
    """代表一隻狗的類。"""
    
    def bark(self):
        """狗叫的行為。"""
        return "汪汪!"

print(Dog.__doc__)  # 輸出：代表一隻狗的類。
print(Dog.bark.__doc__)  # 輸出：狗叫的行為。
```

## 解釋
使用 `__doc__` 時，有幾個常見的陷阱需要注意：
- 確保你的文檔字符串位於對象的第一行，如果不在，`__doc__` 將不會包含該字符串。
- 文檔字符串應該清楚明瞭，簡潔地描述對象的功能。
- 如果對象沒有文檔字符串，調用 `__doc__` 會返回 `None`，這可能會導致調試時的困惑。

## 一句話總結
`__doc__` 屬性提供了對 Python 對象的文檔字符串的訪問，幫助開發者理解和使用代碼。
