# Python의 __doc__ 속성: 문서화의 핵심

## 개요
Python의 `__doc__` 속성은 모듈, 클래스, 함수, 메서드에 대한 문서 문자열을 반환하며, 코드의 가독성을 높이고 사용자를 위한 유용한 정보를 제공합니다. 이 속성은 개발자가 작성한 문서화된 내용을 쉽게 찾고 활용할 수 있게 해줍니다.

## 문서화
`__doc__` 속성은 Python 객체의 문서 문자열을 담고 있습니다. 문서 문자열은 일반적으로 객체의 정의 바로 아래에 위치하며, 삼중 따옴표(`'''` 또는 `"""`)로 감싸져 있습니다. 이 속성을 통해 객체에 대한 설명을 조회할 수 있으며, 이는 코드의 이해를 돕는 중요한 역할을 합니다.

### 사용법
- 객체의 `__doc__` 속성에 접근하려면, 해당 객체 뒤에 `. __doc__`를 사용합니다.
- 예를 들어, 함수나 클래스의 정의 후, 해당 객체의 `__doc__` 속성을 호출하면 그에 대한 설명을 반환합니다.

### 세부사항
- `__doc__` 속성은 문자열 형태로 반환되며, 객체가 문서 문자열을 가지고 있지 않은 경우에는 `None`을 반환합니다.
- 문서 문자열은 코드의 주석을 대신할 수 있으며, 사용자가 해당 기능을 이해하는 데 도움을 줍니다.
- PEP 257은 Python에서 문서 문자열을 작성하는 규칙을 제시하고 있습니다.

## 예제
```python
def add(a, b):
    """두 숫자의 합을 반환합니다."""
    return a + b

print(add.__doc__)  # 출력: 두 숫자의 합을 반환합니다.
```

```python
class MyClass:
    """이 클래스는 예시 클래스를 나타냅니다."""
    
    def method(self):
        """이 메서드는 예시 메서드를 나타냅니다."""
        pass

print(MyClass.__doc__)  # 출력: 이 클래스는 예시 클래스를 나타냅니다.
print(MyClass.method.__doc__)  # 출력: 이 메서드는 예시 메서드를 나타냅니다.
```

## 설명
- `__doc__` 속성은 문서화를 위한 중요한 도구입니다. 그러나 모든 객체가 문서 문자열을 갖고 있는 것은 아니므로, 문서 문자열이 없다면 `None`을 반환할 수 있습니다.
- 문서 문자열을 작성할 때는 일관된 형식을 유지하는 것이 중요하며, PEP 257의 규칙을 따르는 것이 좋습니다.
- `__doc__` 속성을 활용하면, 인터랙티브한 Python 셸에서나 IDE에서 코드의 사용법을 쉽게 확인할 수 있습니다.

## 한 줄 요약
Python의 `__doc__` 속성은 객체에 대한 문서 문자열을 반환하여 코드의 가독성을 높이고 이해를 돕는 중요한 기능입니다.
