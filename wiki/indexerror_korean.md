# Python의 IndexError: 인덱스 오류에 대한 이해

## 개요
IndexError는 Python에서 리스트, 튜플, 문자열 등의 시퀀스 타입에서 잘못된 인덱스를 참조할 때 발생하는 예외입니다. 이 오류는 프로그래머가 시퀀스의 길이를 초과하는 인덱스에 접근하려고 할 때 발생합니다.

## 문서화
### 목적
IndexError는 개발자가 시퀀스 데이터를 다룰 때 발생할 수 있는 일반적인 오류로, 데이터의 경계를 넘어선 접근을 방지하고 코드의 안정성을 높이는 데 도움을 줍니다.

### 사용법
IndexError는 다음과 같은 상황에서 발생합니다:

- 리스트의 인덱스를 잘못 지정했을 때
- 문자열의 인덱스를 범위를 초과하여 참조했을 때
- 튜플의 인덱스를 잘못 지정했을 때

이 오류는 `try`와 `except` 문을 사용하여 처리할 수 있습니다.

### 세부사항
- **발생 조건**: 시퀀스의 길이가 n일 때, 유효한 인덱스는 0부터 n-1까지입니다. 만약 이 범위를 벗어난 인덱스를 사용하면 IndexError가 발생합니다.
- **오류 메시지**: 오류 메시지는 대개 "list index out of range" 또는 "string index out of range"와 같은 형식으로 나타납니다.

## 예제
```python
# 리스트에서 유효하지 않은 인덱스 접근 예시
my_list = [1, 2, 3]
print(my_list[3])  # IndexError 발생

# 튜플에서 유효하지 않은 인덱스 접근 예시
my_tuple = (1, 2, 3)
print(my_tuple[5])  # IndexError 발생

# 문자열에서 유효하지 않은 인덱스 접근 예시
my_string = "Hello"
print(my_string[10])  # IndexError 발생
```

## 설명
IndexError는 주로 다음과 같은 실수로 발생합니다:

1. **인덱스 범위 초과**: 리스트, 튜플, 문자열의 길이보다 큰 인덱스를 사용.
2. **비어 있는 시퀀스에 접근**: 빈 리스트나 튜플에서 인덱스를 사용하려고 할 때 발생.
3. **부정확한 계산**: 인덱스 계산이 잘못되어 유효하지 않은 값이 나올 때.

이러한 오류를 피하기 위해서는 항상 인덱스의 범위를 확인하고, 시퀀스의 길이를 확인하는 것이 좋습니다.

## 한 줄 요약
IndexError는 Python에서 시퀀스의 범위를 초과하는 인덱스 접근 시 발생하는 예외입니다.
