# UnboundLocalError: 파이썬에서 지역 변수 참조 오류

## 개요
`UnboundLocalError`는 파이썬에서 지역 변수가 참조되었지만 초기화되지 않았을 때 발생하는 예외입니다. 이 오류는 주로 변수가 함수 내에서 사용될 때 그 변수가 지역 변수로 인식되지만, 함수 내에서 값을 할당받지 않았을 때 나타납니다.

## 문서
`UnboundLocalError`는 파이썬의 내장 예외 중 하나로, 특정 상황에서 발생하는 오류를 명확히 알려줍니다. 이 오류는 주로 다음과 같은 경우에 발생합니다:

1. **지역 변수의 사용**: 함수 내에서 변수를 선언하고 값을 할당하지 않거나, 이전에 해당 변수를 초기화하지 않고 사용하려 할 때 발생합니다.
2. **전역 변수와의 혼동**: 함수 내에서 전역 변수를 수정하려고 시도할 때, 파이썬은 해당 변수를 지역 변수로 간주하고 값이 할당되지 않은 상태에서 접근하려 할 경우 `UnboundLocalError`가 발생합니다.

### 사용법
`UnboundLocalError`를 피하기 위해서는 변수를 함수 내에서 사용하기 전에 반드시 초기화해야 하며, 필요시 `global` 키워드를 사용하여 전역 변수를 사용할 수 있습니다.

## 예제
아래는 `UnboundLocalError`가 발생하는 간단한 예제입니다:

```python
def example_function():
    print(x)  # UnboundLocalError 발생
    x = 10

example_function()
```

위의 코드에서 `x`는 함수 내에서 초기화되지 않았기 때문에 `UnboundLocalError`가 발생합니다.

### 전역 변수를 사용할 경우의 예
전역 변수를 사용하고자 할 경우, 다음과 같이 `global` 키워드를 사용할 수 있습니다:

```python
x = 5

def example_function():
    global x
    print(x)  # 5 출력
    x = 10

example_function()
print(x)  # 10 출력
```

## 설명
`UnboundLocalError`는 다음과 같은 일반적인 착오로 인해 발생할 수 있습니다:

- **변수 초기화 누락**: 변수를 정의하기 전에 사용하면 항상 이 오류가 발생합니다.
- **전역 변수와 지역 변수 혼용**: 함수 내에서 이미 존재하는 전역 변수를 수정하고자 할 때 `global` 키워드를 사용하지 않으면, 해당 변수는 지역 변수로 간주되어 오류가 발생합니다.

이 오류를 해결하려면, 변수를 사용하기 전에 반드시 초기화하거나, 전역 변수를 사용할 경우 `global` 키워드를 명시적으로 선언해야 합니다.

## 한 줄 요약
`UnboundLocalError`는 파이썬에서 지역 변수가 초기화되지 않은 상태에서 참조될 때 발생하는 예외입니다.
