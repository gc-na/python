# Python의 __name__ 변수: 이해와 활용

## 개요
`__name__`은 Python에서 모듈의 이름을 나타내는 특별한 변수입니다. 이 변수는 스크립트가 직접 실행될 때와 다른 모듈에 의해 임포트될 때의 동작을 구분하는 데 유용합니다.

## 문서화
`__name__` 변수는 Python 인터프리터가 모듈을 실행할 때 자동으로 설정되는 내장 변수입니다. 스크립트가 직접 실행될 때 `__name__`은 `"__main__"`으로 설정되며, 다른 모듈에 의해 임포트될 경우 해당 모듈의 이름으로 설정됩니다. 이 기능은 코드의 재사용성을 높이고, 특정 코드 블록을 모듈로서 임포트할 때는 실행하지 않도록 하는 데 유용합니다.

### 용도
- 모듈이 직접 실행되는지 또는 다른 모듈에 의해 임포트되고 있는지를 확인하기 위해 사용됩니다.
- 스크립트의 테스트 코드나 예제 코드를 모듈 로직과 분리할 수 있습니다.

### 사용법
다음은 `__name__`을 사용하는 기본적인 방법입니다:

```python
if __name__ == "__main__":
    # 이 블록은 스크립트가 직접 실행될 때만 실행됩니다.
    print("이 스크립트는 직접 실행되고 있습니다.")
else:
    print("이 스크립트는 다른 모듈에 의해 임포트되었습니다.")
```

## 예제
```python
# my_module.py
def greet():
    print("안녕하세요!")

if __name__ == "__main__":
    greet()
```

- `my_module.py`를 직접 실행하면 "안녕하세요!"가 출력됩니다.
- 다른 파일에서 `import my_module`을 사용하면 "안녕하세요!"는 출력되지 않습니다.

## 설명
`__name__` 변수는 매우 유용하지만, 몇 가지 주의해야 할 점이 있습니다:
- `__name__`을 잘못 사용하면 예상치 못한 결과를 초래할 수 있습니다. 예를 들어, 메인 스크립트에서 실행할 때와 임포트할 때의 동작을 혼동하면 안 됩니다.
- 모든 Python 파일에는 기본적으로 `__name__` 변수가 존재합니다. 따라서 이 변수를 사용하여 코드의 흐름을 제어하는 것이 가능합니다.

## 한 줄 요약
`__name__` 변수는 Python에서 모듈의 실행 컨텍스트를 식별하는 데 사용되는 특별한 변수로, 스크립트가 직접 실행될 때와 임포트될 때의 동작을 구분합니다.
