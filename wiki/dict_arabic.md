# القاموس في بايثون: كل ما تحتاج معرفته عن dict

## الملخص
تعتبر dict (القاموس) واحدة من الهياكل الأساسية في لغة البرمجة بايثون، وتستخدم لتخزين البيانات في شكل أزواج من المفاتيح والقيم، مما يسمح بالوصول السريع إلى العناصر.

## الوثائق
### الغرض
القاموس في بايثون هو نوع بيانات غير مرتب يستخدم لتخزين البيانات في شكل أزواج من المفاتيح والقيم. كل مفتاح يجب أن يكون فريدًا، ويمكن أن تكون القيم من أي نوع بيانات متاح في بايثون.

### الاستخدام
لإنشاء قاموس جديد، يمكن استخدام الأقواس المعقوفة `{}` أو دالة `dict()`. يمكن إضافة أو تعديل القيم باستخدام المفاتيح، كما يمكن حذف العناصر باستخدام كلمة `del`.

### التفاصيل
- **إنشاء قاموس**:
  ```python
  my_dict = {'name': 'Ali', 'age': 30}
  ```
  أو:
  ```python
  my_dict = dict(name='Ali', age=30)
  ```

- **الوصول إلى القيم**:
  للوصول إلى قيمة معينة، استخدم المفتاح:
  ```python
  print(my_dict['name'])  # يطبع: Ali
  ```

- **تعديل القيم**:
  لتعديل قيمة موجودة:
  ```python
  my_dict['age'] = 31
  ```

- **إضافة عناصر**:
  لإضافة عنصر جديد:
  ```python
  my_dict['city'] = 'Riyadh'
  ```

- **حذف عنصر**:
  لحذف عنصر:
  ```python
  del my_dict['age']
  ```

## الأمثلة
### مثال 1: إنشاء قاموس وطباعة القيم
```python
person = {'name': 'Sara', 'age': 25}
print(person['name'])  # يطبع: Sara
```

### مثال 2: تعديل وإضافة عناصر
```python
person['age'] = 26  # تعديل العمر
person['city'] = 'Cairo'  # إضافة المدينة
print(person)  # يطبع: {'name': 'Sara', 'age': 26, 'city': 'Cairo'}
```

### مثال 3: حذف عنصر
```python
del person['city']  # حذف المدينة
print(person)  # يطبع: {'name': 'Sara', 'age': 26}
```

## الشرح
### الأخطاء الشائعة
- **استخدام مفاتيح مكررة**: إذا تم استخدام مفتاح مكرر عند إنشاء القاموس، سيتم استبدال القيمة القديمة بالقيمة الجديدة.
- **محاولة الوصول إلى مفتاح غير موجود**: إذا حاولت الوصول إلى مفتاح غير موجود، ستظهر لك خطأ `KeyError`. لتجنب ذلك، يمكنك استخدام `get()`:
  ```python
  value = person.get('city', 'غير موجود')  # سيعيد 'غير موجود' إذا لم يكن المفتاح موجودًا
  ```

### ملاحظات إضافية
- القاموس غير مرتب في الإصدارات السابقة من بايثون (مثل 3.6 وما قبل)، لكن في الإصدارات الأحدث (بايثون 3.7 وما بعد) أصبح يحتفظ بترتيب إدخال العناصر.
- يمكن استخدام القواميس كقيم في القواميس الأخرى، مما يسمح بإنشاء هياكل بيانات معقدة.

## ملخص في جملة واحدة
القاموس في بايثون هو هيكل بيانات قوي يتيح تخزين البيانات في شكل أزواج من المفاتيح والقيم، مما يسهل الوصول إليها وإدارتها.
