# Python exec 함수: 동적 코드 실행의 모든 것

## 개요
`exec`는 Python에서 문자열로 된 코드를 실행할 수 있는 강력한 내장 함수입니다. 이 기능은 동적으로 생성되는 코드를 실행해야 하는 상황에서 유용하게 사용됩니다.

## 문서화
### 목적
`exec` 함수는 주어진 문자열을 Python 코드로 해석하고 실행합니다. 이를 통해 런타임에서 동적으로 코드를 생성하고 실행할 수 있습니다.

### 사용법
`exec` 함수의 기본 구문은 다음과 같습니다:

```python
exec(object[, globals[, locals]])
```

- `object`: 실행할 코드 블록으로, 문자열이나 코드 객체 형태로 제공됩니다.
- `globals`: (선택적) 전역 네임스페이스를 나타내는 딕셔너리입니다. 기본값은 현재의 전역 네임스페이스입니다.
- `locals`: (선택적) 지역 네임스페이스를 나타내는 딕셔너리입니다. 기본값은 현재의 지역 네임스페이스입니다.

### 세부사항
`exec`는 주로 동적 코드 실행에 사용되며, 보안상의 이유로 신뢰할 수 없는 소스에서 실행하는 것은 피해야 합니다. 또한, `exec`로 실행된 코드는 현재의 네임스페이스에 영향을 미치므로, 변수나 함수가 예상치 못하게 덮어쓰일 수 있습니다.

## 예제
### 기본 사용 예
```python
code = 'print("Hello, World!")'
exec(code)
```
이 코드는 "Hello, World!"를 출력합니다.

### 변수 할당 예
```python
code = '''
x = 10
y = 20
result = x + y
'''
exec(code)
print(result)  # 30
```
여기서는 `x`, `y`, `result` 변수를 정의하고, `exec`를 통해 계산한 결과를 출력합니다.

## 설명
`exec` 사용 시 주의해야 할 점은 다음과 같습니다:

1. **보안 문제**: `exec`로 실행되는 코드는 악의적인 코드가 포함될 수 있으므로, 신뢰할 수 없는 소스에서 입력된 코드를 실행하는 것은 위험합니다.
2. **네임스페이스 오염**: `exec`로 실행한 코드가 현재 스코프의 변수와 함수에 영향을 줄 수 있습니다. 따라서, 다른 코드와 충돌할 수 있는 변수 이름을 사용하지 않도록 주의해야 합니다.
3. **디버깅의 어려움**: 동적으로 생성된 코드는 디버깅이 어려울 수 있으며, 코드의 흐름을 추적하기 힘들 수 있습니다.

## 한 줄 요약
Python의 `exec` 함수는 문자열로 된 코드를 실행하여 동적 프로그래밍을 가능하게 하지만, 보안과 네임스페이스 관리에 주의해야 합니다.
