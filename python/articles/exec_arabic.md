# exec في بايثون: استخدامه وأهميته

## ملخص
تُعتبر دالة `exec` في بايثون أداة قوية لتنفيذ كود بايثون المقدم كجملة نصية، مما يتيح للمطورين القدرة على تنفيذ التعليمات البرمجية الديناميكية خلال وقت التشغيل.

## الوثائق
تُستخدم دالة `exec` لتنفيذ كود بايثون مكتوب كنص. يمكن استخدامها لتشغيل أي تعبير بايثون، بما في ذلك تعريف الوظائف، المتغيرات، والهياكل الأخرى. 

### الصيغة
```python
exec(object[, globals[, locals]])
```

- **object**: سلسلة نصية تحتوي على كود بايثون المراد تنفيذه.
- **globals**: (اختياري) قاموس يحدد مساحة الأسماء العالمية، يمكنك تمرير `None` إذا كنت لا ترغب في استخدام مساحة أسماء عالمية محددة.
- **locals**: (اختياري) قاموس يحدد مساحة الأسماء المحلية، يمكنك تمرير `None` إذا كنت لا ترغب في استخدام مساحة أسماء محلية محددة.

### الغرض والاستخدام
تستخدم `exec` لأغراض متعددة مثل:
- تنفيذ كود بايثون الديناميكي.
- إنشاء كود في وقت التشغيل.
- تنفيذ سكربتات أو تعليمات برمجية مخزنة.

## أمثلة
### مثال 1: تنفيذ كود بسيط
```python
code = 'print("Hello, World!")'
exec(code)
```
**الناتج:**
```
Hello, World!
```

### مثال 2: استخدام متغيرات
```python
x = 10
code = 'y = x + 5'
exec(code)
print(y)  # الناتج: 15
```

### مثال 3: تعديل مساحة الأسماء
```python
globals_dict = {}
exec('result = 5 * 2', globals_dict)
print(globals_dict['result'])  # الناتج: 10
```

## الشرح
### المشاكل الشائعة
- **الأمان**: استخدام `exec` يمكن أن يكون خطيرًا إذا تم تنفيذ كود غير موثوق، لأنه يمكن أن يؤدي إلى تنفيذ تعليمات برمجية ضارة. يجب توخي الحذر عند استخدامه مع مدخلات المستخدم.
- **الأداء**: قد تؤثر `exec` سلبًا على الأداء بسبب تحليل النص وتنفيذه في وقت التشغيل.
- **التحكم في النطاق**: من المهم فهم كيفية عمل مساحات الأسماء العالمية والمحلية عند استخدام `exec`، حيث أن المتغيرات يمكن أن تتداخل أو تكون غير مُعرفة.

## ملخص جملة واحدة
تسمح دالة `exec` في بايثون بتنفيذ كود بايثون مكتوب كنص، مما يمنح المطورين القدرة على العمل مع التعليمات البرمجية الديناميكية.
