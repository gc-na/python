# NotADirectoryError: 파이썬에서 디렉토리가 아닌 오류

## 개요
`NotADirectoryError`는 파이썬에서 파일 시스템 작업을 수행할 때 디렉토리로 예상되는 경로가 실제로는 디렉토리가 아닐 때 발생하는 예외입니다. 이 오류는 주로 파일 또는 디렉토리 경로를 처리할 때 발생하며, 잘못된 경로 지정으로 인해 프로그램이 중단되지 않도록 도와줍니다.

## 문서화
`NotADirectoryError`는 파이썬의 내장 예외 클래스 중 하나로, `OSError`의 하위 클래스입니다. 이 예외는 파일 시스템 상에서 디렉토리로 사용해야 할 경로가 파일인 경우에 발생합니다. 일반적으로 `os` 모듈이나 `pathlib` 모듈을 사용할 때 이 오류가 발생할 수 있습니다.

### 용도
- **파일 시스템 작업**: 디렉토리에 파일을 생성하거나 접근할 때 잘못된 경로를 지정하면 이 오류가 발생합니다.
- **디렉토리 확인**: 특정 경로가 디렉토리인지 확인할 때 유용합니다.

### 세부사항
- **발생 조건**: 이 오류는 다음과 같은 상황에서 발생할 수 있습니다:
  - `os.listdir()` 함수로 디렉토리 내용을 읽으려 할 때,
  - `os.makedirs()` 또는 `os.rename()` 등의 함수에서 디렉토리를 기대했으나 파일이 있을 때.
  
- **예외 처리**: `try-except` 블록을 사용하여 이 오류를 처리할 수 있습니다. 이는 프로그램의 흐름을 제어하고, 사용자에게 친절한 오류 메시지를 제공하는 데 유용합니다.

## 예제
```python
import os

# 잘못된 경로로 NotADirectoryError 발생 예제
try:
    # 파일 경로를 지정합니다.
    file_path = 'example.txt'
    # 디렉토리 내용을 읽기 시도
    contents = os.listdir(file_path)
except NotADirectoryError as e:
    print(f"오류 발생: {e}")
```

## 설명
`NotADirectoryError`는 디렉토리 경로를 요구하는 함수에서 파일 경로를 제공할 때 발생합니다. 이 오류에 대한 일반적인 실수는 다음과 같습니다:
- 파일 시스템에서 파일과 디렉토리의 경로를 혼동하는 경우.
- 잘못된 경로 문자열을 사용하여 경로를 지정하는 경우.
  
이는 주로 경로를 하드코딩할 때 발생하므로, 경로를 동적으로 생성하거나 확인하는 것이 좋습니다.

## 한 줄 요약
`NotADirectoryError`는 디렉토리 대신 파일 경로를 제공할 때 발생하는 파이썬 예외입니다.
