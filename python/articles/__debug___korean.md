# Python의 __debug__ 변수: 디버깅 모드와 코드 최적화

## 개요
`__debug__`는 Python에서 제공하는 내장 상수로, 코드가 디버그 모드인지 여부를 나타냅니다. 이 변수는 Python 인터프리터가 최적화 모드에서 실행되고 있는지 확인하는 데 사용됩니다.

## 문서화
### 목적
`__debug__`는 Python 코드에서 디버그 모드가 활성화되어 있는지를 결정하는 데 유용합니다. 디버그 모드에서는 코드의 오류를 쉽게 잡아내기 위해 추가적인 검사와 경고가 활성화됩니다. 반면, 최적화 모드에서는 이러한 검사들이 비활성화되어 코드 성능을 높일 수 있습니다.

### 사용법
- `__debug__`는 Boolean 값으로, 디버그 모드가 활성화되면 `True`, 비활성화되면 `False`로 설정됩니다.
- 일반적으로 `-O` 플래그를 사용하여 Python 스크립트를 실행할 때 디버그 모드를 비활성화할 수 있습니다. 이때 `__debug__`는 `False`가 됩니다.

### 세부 사항
- `__debug__`는 Python의 모든 모듈에서 사용할 수 있으며, 변경할 수 없습니다.
- 디버깅을 위한 개발 중에는 `__debug__`가 `True`인 상태가 일반적입니다.
- 이 변수를 활용하여 코드의 특정 부분에서 디버그 정보를 출력하거나, 디버그 관련 기능을 조건부로 활성화할 수 있습니다.

## 예제
```python
if __debug__:
    print("디버그 모드에서 실행 중입니다.")
else:
    print("최적화 모드에서 실행 중입니다.")
```

```python
def example_function():
    if __debug__:
        print("디버그 정보: 함수가 호출되었습니다.")
    # 함수의 주요 로직
    return True

example_function()
```

## 설명
- `__debug__`를 활용할 때 주의할 점은, 최적화 모드에서 실행할 경우 디버그 관련 코드가 비활성화된다는 것입니다. 이로 인해 디버그 정보를 놓칠 수 있습니다.
- 코드가 최적화되면 assert 문과 같은 디버그 관련 기능이 무시되므로, 배포 전 최적화 모드에서의 테스트가 중요합니다.
- 또한, `__debug__`는 전역에서 사용할 수 있지만, 조건부로 코드 블록을 작성할 때 주의가 필요합니다.

## 한 줄 요약
`__debug__`는 Python에서 코드가 디버그 모드인지 여부를 나타내는 내장 상수로, 최적화된 실행과 디버깅 정보를 관리하는 데 사용됩니다.
