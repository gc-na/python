# القابلية للتنفيذ في بايثون (Callable في Python)

## ملخص
القابلية للتنفيذ (Callable) في بايثون تشير إلى أي كائن يمكن استدعاؤه كدالة. تشمل هذه الكائنات الدوال، الكائنات القابلة للتنفيذ، وطرق الكائنات. يتيح لك ذلك استخدام الدوال كأولويات في التعليمات البرمجية، مما يعزز من مرونة البرمجة.

## التوثيق
### الغرض
الهدف من مفهوم القابلية للتنفيذ هو السماح للمطورين بإنشاء كائنات يمكن استدعاؤها، مما يسهل كتابة التعليمات البرمجية الديناميكية وتطوير التطبيقات المعقدة.

### الاستخدام
في بايثون، يمكن التحقق مما إذا كان الكائن قابلًا للتنفيذ باستخدام الدالة المدمجة `callable()`. تأخذ هذه الدالة كائنًا كمعامل وتعيد قيمة `True` إذا كان الكائن قابلًا للتنفيذ، و`False` إذا لم يكن كذلك.

### التفاصيل
- **الدوال**: كل دالة معرفة في بايثون هي كائن قابل للتنفيذ.
- **الكائنات القابلة للتنفيذ**: يمكن أيضًا تعريف كائنات قابلة للتنفيذ عن طريق تعريف الدالة `__call__()` داخل فئة.
- **الطرق**: جميع الطرق (methods) المرتبطة بكائنات هي أيضًا قابلة للتنفيذ.

### الصيغة
```python
callable(object)
```

## أمثلة
### مثال 1: استخدام الدوال
```python
def my_function():
    return "Hello, World!"

print(callable(my_function))  # النتيجة: True
```

### مثال 2: استخدام الكائنات القابلة للتنفيذ
```python
class MyCallable:
    def __call__(self):
        return "I am callable!"

obj = MyCallable()
print(callable(obj))  # النتيجة: True
```

### مثال 3: الطرق
```python
class MyClass:
    def my_method(self):
        return "Method called!"

obj = MyClass()
print(callable(obj.my_method))  # النتيجة: True
```

## الشرح
### الأخطاء الشائعة
- **فهم غير صحيح للمصطلح**: قد يخلط البعض بين القابلية للتنفيذ وخصائص أخرى مثل القابلية للتكرار (iterable).
- **الكائنات غير القابلة للتنفيذ**: إذا حاولت استدعاء كائن غير قابل للتنفيذ، ستظهر لك خطأ `TypeError`.
- **تجاهل الدوال المجهولة**: الدوال التي لا تحمل اسمًا (anonymous functions) مثل دوال lambda يمكن أن تكون أيضًا قابلة للتنفيذ.

### ملاحظات إضافية
- يمكن استخدام الدوال كوسائط للدوال الأخرى، مما يعزز من إمكانية استخدام القابلية للتنفيذ في العديد من سياقات البرمجة.
- تأكد من أن الكائنات التي تستخدمها كدوال قد تم تعريفها بشكل صحيح.
