# PythonのNameErrorについて - エラー処理とトラブルシューティング

## 概要
Pythonにおける`NameError`は、アクセスしようとした変数や関数が定義されていない場合に発生するエラーです。このエラーは、プログラムのデバッグ時に頻繁に遭遇するもので、初心者から上級者まで注意が必要です。

## ドキュメント
`NameError`は、Pythonの例外の一つであり、特定の名前がローカルまたはグローバルスコープで見つからない場合に発生します。主に以下のような状況で発生します。

- 定義されていない変数や関数を参照したとき
- スコープの外にある変数を参照したとき
- タイプミスによって、変数の名前が間違っているとき

### 使用法
`NameError`は通常、プログラムの実行中に自動的に発生します。特に、コードを記述している際に変数や関数の名前を誤って記述すると、このエラーが発生します。

```python
print(x)  # xは定義されていないため、NameErrorが発生します。
```

## 例
以下に、`NameError`の基本的な使用例を示します。

```python
# 定義されていない変数へのアクセス
print(a)  # NameError: name 'a' is not defined

# タイプミスによるエラー
b = 10
print(B)  # NameError: name 'B' is not defined
```

## 説明
`NameError`は、特にスコープや変数の名前に関して注意が必要です。以下は、よくある落とし穴や注意点です。

- **スコープの理解**：変数は定義されたスコープ内でのみアクセス可能です。例えば、関数内で定義された変数は、その関数の外からはアクセスできません。
  
```python
def my_function():
    c = 5

my_function()
print(c)  # NameError: name 'c' is not defined
```

- **タイプミス**：変数名の大文字と小文字の違いにも注意が必要です。Pythonは大文字と小文字を区別します。

- **未定義の関数やクラス**：関数やクラスを呼び出す際に、事前に定義されていないと`NameError`が発生します。

## 一文要約
`NameError`は、Pythonで未定義の変数や関数にアクセスしようとした際に発生するエラーです。
