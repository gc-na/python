# خطأ الصياغة (SyntaxError) في بايثون: دليل شامل

## ملخص
خطأ الصياغة (SyntaxError) في بايثون هو أحد الأخطاء الأساسية التي تحدث عندما تكون الشيفرة البرمجية مكتوبة بطريقة غير صحيحة من الناحية النحوية. يُشير هذا الخطأ إلى أن بايثون لم يتمكن من فهم الشيفرة بسبب عدم توافقها مع قواعد اللغة.

## الوثائق
### الغرض
يهدف خطأ الصياغة إلى تنبيه المطورين إلى وجود مشكلة في الشيفرة البرمجية، مما يمنع تنفيذ البرنامج حتى يتم تصحيح الخطأ.

### الاستخدام
يظهر خطأ الصياغة عادةً عند وجود:
- عدم توافق الأقواس أو علامات الترقيم.
- استخدام كلمات محجوزة بطريقة غير صحيحة.
- كتابة التعليمات البرمجية بدون المسافات البادئة المطلوبة.

### التفاصيل
عند حدوث خطأ صياغي، سيقوم المترجم بإظهار رسالة توضح موقع الخطأ، بالإضافة إلى نوع الخطأ. تتضمن الرسالة عادةً رقم السطر الذي حدث فيه الخطأ ونوع الخطأ نفسه.

## أمثلة
### مثال 1: أقواس غير متطابقة
```python
print("Hello, World!"
```
النتيجة:
```
SyntaxError: unexpected EOF while parsing
```

### مثال 2: استخدام كلمة محجوزة
```python
def = 5
```
النتيجة:
```
SyntaxError: invalid syntax
```

### مثال 3: عدم وجود مسافات بادئة
```python
if True:
print("This will cause a SyntaxError")
```
النتيجة:
```
IndentationError: expected an indented block
```

## الشرح
من الشائع أن يقع المطورون في أخطاء نحوية بسيطة مثل نسيان إغلاق قوس أو كتابة كلمة محجوزة بشكل غير صحيح. من المهم قراءة رسالة الخطأ بعناية لتحديد موقع المشكلة. يمكن أن يؤدي وجود مسافات بادئة غير صحيحة إلى ظهور خطأ مختلف يُعرف باسم "خطأ في المسافات البادئة" (IndentationError)، وهو نوع خاص من أخطاء الصياغة.

## ملخص من جملة واحدة
خطأ الصياغة (SyntaxError) في بايثون يُشير إلى وجود مشكلة في الشيفرة البرمجية تمنع البرنامج من التنفيذ بسبب عدم توافقه مع قواعد اللغة.
