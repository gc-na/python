# Python 中的例外處理 (Exception)

## 概述
在 Python 中，例外是一種事件，當程序執行過程中出現錯誤時會被觸發。例外處理機制允許開發者捕獲並處理這些錯誤，以避免程序崩潰並提供更好的用戶體驗。

## 文檔

### 目的
例外處理的主要目的是提高程序的穩定性和可用性。通過捕獲和處理錯誤，開發者可以控制程序在面對異常情況時的行為，從而避免不必要的程序崩潰。

### 使用方式
在 Python 中，例外處理主要使用 `try`、`except`、`else` 和 `finally` 語句來實現。以下是這些語句的基本用法：

- `try`：用於包裹可能引發例外的代碼區塊。
- `except`：用於捕獲並處理特定的例外。
- `else`：可選的，當 `try` 區塊沒有引發例外時執行。
- `finally`：可選的，無論是否引發例外，該區塊的代碼都會執行。

### 例子
以下是 Python 中例外處理的基本用法示例：

```python
try:
    # 嘗試執行的代碼
    result = 10 / 0  # 故意引發 ZeroDivisionError
except ZeroDivisionError:
    print("錯誤：不能除以零！")
else:
    print("計算結果是：", result)
finally:
    print("這段代碼總是會執行。")
```

輸出結果：
```
錯誤：不能除以零！
這段代碼總是會執行。
```

## 解釋
在使用例外處理時，有一些常見的陷阱和注意事項：

- **未捕獲的例外**：如果沒有適當的 `except` 區塊來捕獲特定的例外，程序將崩潰並顯示錯誤信息。
- **多個例外處理器**：可以使用多個 `except` 區塊來處理不同類型的例外，也可以使用元組來同時捕獲多種例外。
- **自定義例外**：可以創建自定義例外類別，以便更好地處理特定的錯誤情況。

```python
class MyError(Exception):
    pass

try:
    raise MyError("這是一個自定義的錯誤！")
except MyError as e:
    print(e)
```

輸出結果：
```
這是一個自定義的錯誤！
```

## 一行總結
Python 的例外處理機制透過 `try`、`except` 等關鍵字，幫助開發者有效捕獲和處理運行時錯誤，提升程序的穩定性。
